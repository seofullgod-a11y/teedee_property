คู่มือการใช้งาน

อยากได้ไฟล์ไว้แชร์ทีมหรือพิมพ์? ดาวน์โหลด Word

1. การเข้าใช้งานหลังบ้าน

  1. เปิดเบราว์เซอร์ไปที่ yoojai.com/admin แล้วกรอกรหัสผ่านทีมงาน
  2. เมนูทั้งหมดอยู่แถบด้านซ้าย (บนมือถือ: ปุ่มเมนูมุมบน)
⚠ ระบบจำกัดการลองรหัสผิดไม่เกิน 8 ครั้งต่อ 15 นาที — พิมพ์ผิดหลายครั้งให้รอสักครู่ อย่ากดรัว
💡 ติดตั้งเป็นแอปบนมือถือได้: เปิดเว็บ → เมนูเบราว์เซอร์ → "เพิ่มลงหน้าจอหลัก"

2. การลงประกาศทรัพย์ (ทีละรายการ)

2.1 ชื่อประกาศ

  • เขียนให้เห็นภาพใน 1 บรรทัด: ประเภท + จุดเด่น + ทำเล เช่น "คอนโด 1 นอน วิวแม่น้ำ ใกล้ BTS กรุงธนบุรี 300 ม."
  • หลีกเลี่ยงตัวพิมพ์ใหญ่ทั้งหมด/เครื่องหมายตกใจเยอะ ๆ — ดูเป็นสแปม

2.2 รูปภาพ (สำคัญมาก)

  • ใส่อย่างน้อย 3–5 รูปต่อรายการ รูปแรกคือรูปปก — เลือกมุมสวยสุด แสงดีสุด
  • วางลิงก์รูป (URL) ทีละบรรทัด แนะนำรูปแนวนอน (4:3 หรือ 16:9)

2.3 ตำแหน่งบนแผนที่

  • เปิด Google Maps → ปักหมุดที่ตัวบ้าน → กด "แชร์" → คัดลอกลิงก์ → วางในช่อง "ตำแหน่ง Google Maps"
  • ระบบจะแกะพิกัดให้อัตโนมัติ หมุดตรงหลังเป๊ะ — ถ้าเว้นว่างจะปักโดยประมาณตามจังหวัด

2.4 ผู้ดูแลและป้ายพิเศษ

  • เลือกผู้ดูแลทรัพย์ทุกครั้ง — ชื่อและช่องทางติดต่อจะแสดงบนหน้าทรัพย์
  • ทรัพย์เด่นติ๊ก "ประกาศแนะนำ" (ขึ้นหน้าแรก) หรือเลือกป้าย เช่น "มาแรง"
💡 กดปุ่ม AI ช่วยเขียนคำโปรยได้ — ใส่โน้ตสั้น ๆ เช่น "เพิ่งรีโนเวท เจ้าของใจดี" แล้วให้ AI เรียบเรียง

3. การลงทรัพย์ทีละหลายรายการ (นำเข้า CSV)

  1. เมนู "นำเข้า / ตัวอย่าง" → "ดาวน์โหลดเทมเพลต CSV"
  2. เปิดด้วย Excel หรือ Google Sheets (มีตัวอย่าง 2 แถวให้ดู)
  3. กรอกข้อมูลจริง — หัวตารางใช้ไทยหรืออังกฤษก็ได้
  4. ช่อง images: วางลิงก์รูปหลายรูปคั่นด้วย | (เส้นตั้ง)
  5. ช่อง map_url: วางลิงก์ Google Maps ของแต่ละหลัง
  6. บันทึกเป็น .csv → อัปโหลด → ตรวจตัวอย่างที่ระบบแสดง
  7. แนะนำ "ไม่ติ๊ก" เผยแพร่ทันที → เก็บเป็นร่างให้ตรวจก่อน
⚠ ราคากรอกเป็นตัวเลขล้วน เช่น 3500000 (ไม่ต้องใส่คอมมาหรือคำว่าบาท)

4. การดูแลลูกค้า (สำคัญที่สุด)

  • ข้อความติดต่อ — รวมทุกคนที่ทักมา พร้อมเบอร์/LINE มีแจ้งเตือนเข้า Telegram ควรตอบใน 1–2 ชั่วโมง
  • นัดชมทรัพย์ — โทรยืนยันแล้วอัปเดตสถานะ: รอยืนยัน → ยืนยันแล้ว → เสร็จสิ้น
  • รีวิว — ลบได้เฉพาะรีวิวไม่เหมาะสม (คำหยาบ/สแปม) ไม่ควรลบรีวิวติเพื่อก่อ

5. อ่านภาพรวมธุรกิจ

  • หน้าแรกหลังบ้านสรุป: ประกาศที่เผยแพร่, มูลค่าพอร์ต, ผู้ติดต่อเดือนนี้, นัดชมค้าง, คะแนนรีวิว
  • เมนู "สถิติผู้ใช้" → "ทรัพย์ที่ควรปรับปรุง" = ยอดดูน้อย/ไม่มีรูป ควรเพิ่มรูปหรือปรับชื่อ
  • ปุ่ม "ส่งออกข้อมูล (CSV)" — ดาวน์โหลดประกาศ/ผู้ติดต่อ/นัดชมเป็น Excel

6. คำถามพบบ่อย

  • ลงประกาศแล้วไม่ขึ้น? เช็คสถานะเป็น "เผยแพร่" ไม่ใช่ "ร่าง" แล้วรีเฟรช
  • หมุดไม่ตรงบ้าน? แก้ไขประกาศ วางลิงก์ Google Maps ของหลังนั้น
  • รูปไม่แสดง? ลิงก์ต้องเปิดดูรูปได้ตรง ๆ (ลงท้าย .jpg .png .webp)
  • เผลอลบประกาศ? แจ้งผู้ดูแลระบบทันที (กู้ได้จาก backup รายวัน)
ไฟล์ Word ฉบับเต็ม (มีตารางตรวจรับให้ติ๊ก) ดาวน์โหลด Word

1. สำรองข้อมูล (ทำครั้งเดียว อุ่นใจตลอด)

  • Railway Backup: เข้า Railway → service ฐานข้อมูล → แท็บ "Backups" → เปิด Daily backups
  • สำรองด้วยมือ (เดือนละครั้ง): ใช้ปุ่ม "ส่งออกข้อมูล (CSV)" ใน Dashboard เก็บใส่ Google Drive บริษัท
💡 ตั้งเตือนปฏิทินทุกวันที่ 1: "ดาวน์โหลด CSV สำรองข้อมูล" ใช้เวลาไม่ถึง 2 นาที

2. เฝ้าระวังเว็บล่ม (ฟรี)

  1. สมัคร uptimerobot.com (แผนฟรีพอ)
  2. Add New Monitor → ชนิด HTTP(s)
  3. URL ใส่: https://yoojai.com/healthz
  4. Interval 5 นาที → ใส่อีเมลรับแจ้งเตือน → Save

ถ้าเว็บเข้าไม่ได้เกิน 5 นาที จะมีอีเมลเตือนทันที ไม่ต้องรอลูกค้าทักมาบอก

3. Deploy รวบยอด (v48 + ความปลอดภัย + ความเร็ว)

  • แนะนำอัปโหลด "ทั้งโฟลเดอร์จาก zip" ทับของเดิม — กันไฟล์ผสมเวอร์ชัน
  • จุดห้ามตกหล่น: package.json + package-lock.json (มี "compression" ใหม่ ถ้าไม่อัปเซิร์ฟเวอร์บูตไม่ขึ้น)
  • รอ Railway build จน Success → ดู Logs เห็น "TeeDee running on :3000"

4. ตรวจรับหลัง Deploy (เปิดแบบ hard refresh)

  • หน้า /about /contact /privacy /terms มี navbar + footer ครบ
  • สลับ Dark Mode → เข้า /admin อ่านได้ชัด
  • กด EN → footer, /discover, ตัวกรอง /search แปลครบ
  • ใส่รหัสแอดมินผิด 9 ครั้ง → ครั้งที่ 9 ขึ้นให้รอ (rate limit)
  • /map มีหมุดครบ · /discover ปัดได้ · ปุ่ม AI เปิดแชทค้าง
  • พิมพ์ URL มั่ว → เจอหน้า 404 · นำเข้า CSV ได้ · กดติดต่อ → เข้า Telegram

5. Environment Variables ที่ควรมีครบ

  • DATABASE_URL — Railway ใส่ให้อัตโนมัติ
  • ADMIN_PASSWORD — ตั้งยาว 12+ ตัว ผสมตัวเลข/สัญลักษณ์
  • BASE_URL — โดเมนจริง เช่น https://yoojai.com (มีผลกับ SEO/ลิงก์แชร์)
  • ANTHROPIC_API_KEY — สำหรับ AI
  • TELEGRAM_BOT_TOKEN + TELEGRAM_CHAT_ID — แจ้งเตือนลูกค้า
  • GOOGLE_SITE_VERIFICATION — ยืนยัน Google Search Console

ตั้งค่าเว็บไซต์

คลิกเพื่อเลือกไฟล์ หรือลากรูปมาวางตรงนี้ PNG · JPG · WEBP · SVG — พื้นหลังโปร่งจะสวยที่สุด (ไม่เกิน ~1.5MB)

โลโก้จะแสดงบน navbar และ footer แทนไอคอนบ้าน — เว้นว่างเพื่อกลับไปใช้ไอคอนเริ่มต้น

อัปโหลดโลโก้เวอร์ชันสีอ่อน สำหรับพื้นหลังมืด ถ้าไม่อัปโหลด ระบบจะใช้โลโก้หลักทั้งสองโหมด
